🌐 Part 6: How to Publish & Share Online (GitHub Pages)

Building the house locally is great, but nobody can visit. GitHub Pages acts as a free road and signboard — giving your guide a live URL anyone can open in their browser.
Step 1: Preparation (Two Critical Rules!)
⚠️ Rule 1: You need an index.html file
GitHub's web server only recognizes one specific filename as the homepage: index.html. If your main file has any other name, visitors will see a blank page or a file listing instead of your guide.

Good news: you already have an index.html file that auto-redirects to the guide — just upload it alongside everything else.
⚠️ Rule 2: Images must be uploaded together with the HTML
All image files (image_5b5c6f.png, etc.) must be uploaded into the same folder level as the HTML file. If you only upload the HTML, all images will appear broken on the live site.
Step 2: Create a GitHub Account (skip if you have one)
  1. Go to github.com and click Sign up in the top right.
  2. Enter your email, choose a password and username, then complete the verification.
  3. The whole process takes about 3 minutes and is completely free.
Step 3: Create a New Repository (your website's storage space)
  1. Once logged in, click the ➕ plus icon top right and select New repository.
  2. In the Repository name field, type a name — e.g., claudecodeinstallguide (letters, numbers, and hyphens only).
  3. Critical: Set visibility to Public. If you choose Private, your friends cannot access the link.
  4. Click the green Create repository button.
💡 Your repository name becomes part of your URL!
A repo named claudecodeinstallguide gives you the link:
https://your-username.github.io/claudecodeinstallguide/
Step 4: Upload All Files
  1. After creating the repo, look for the blue link that says "uploading an existing file" and click it.
  2. Open your D:\claudeai folder, select all of these files:
    • index.html
    • claude_code_guide_v2.html
    • All 8 image files (image_5b5c6f.png, etc.)
    • README.md (optional — shows as a description on the repo page)
  3. Drag all selected files together into the dotted upload area in the browser.
  4. Wait for all files to finish uploading (may take 1–2 minutes).
  5. Scroll to the bottom of the page and click the green Commit changes button to save everything to the cloud.

Figure 9: After uploading, your repository will show all your files listed. Once confirmed, click Commit changes to save everything to the cloud.

⚠️ Warning: If your repository shows Private, your friends cannot access the link. Make sure to set it to Public when creating it.

Step 5: Enable GitHub Pages & Get Your Shareable Link
  1. Go back to your repository's main page and click the Settings tab at the top.
  2. On the left sidebar, scroll down and click Pages.
  3. Under Build and deployment → Branch, change the dropdown from "None" to main, keep the folder as / (root), then click Save.
  4. Wait about 1–2 minutes, then refresh the page.
  5. A green banner will appear at the top saying "Your site is live at..." followed by your unique URL.

⚠️ If you see this upgrade screen, your repository is private and GitHub Pages cannot be enabled. Go back to repository Settings and change visibility to Public to fix this.

Figure 10: The blue box showing "Your site is live at https://skyisthelimit111.github.io/claudecodeinstallguide/" means your site is online. Copy that link and send it to your friends!

✅ Done! How to share the link:
Copy the live URL and send it via WeChat, SMS, or email. Your friend needs no apps or downloads — they just open it in any browser on their phone or computer!

🌐 第六部分：如何发布并生成网页链接发给朋友 (GitHub Pages)

房子在本地盖得再好，别人也看不见。我们需要通过 GitHub 修一条路、挂个门牌号，让朋友们通过手机浏览器就能直接参观。
步骤 1：准备工作 (两项致命注意事项！)
⚠️ 注意事项 1：必须有 index.html 文件
GitHub 网站服务只认识一个固定的"首页"文件名，叫做 index.html。如果你上传的文件叫其他名字（比如 claude_code_guide_v2.html），朋友打开你的链接只会看到一片空白或文件列表。

好消息：你已经有了一个 index.html 文件（自动跳转到你的指南），直接上传它即可。
⚠️ 注意事项 2：图片和 HTML 必须在同一个文件夹
上传时，你的所有图片文件（image_5b5c6f.png 等）必须和 HTML 文件一起上传，放在同一层级。
不能只上传 HTML 文件，否则图片会全部裂开显示不出来。
步骤 2：注册 GitHub 账号（已有账号跳过）
  1. 打开浏览器，访问 github.com，点击右上角 Sign up（注册）。
  2. 填写邮箱、密码、用户名，完成验证后登录。
  3. 整个注册过程约 3 分钟，全程免费。
步骤 3：新建仓库（你的网站储存空间）
  1. 登录后，点击右上角的 ➕ 加号，选择 New repository（新建仓库）。
  2. 在 Repository name（仓库名称） 框里填入名字，例如：claudecodeinstallguide（只能用英文字母、数字和横线）。
  3. 重要： 必须选择 Public（公开），不能选 Private（私有），否则朋友无法访问。
  4. 点击绿色按钮 Create repository（创建仓库）。
💡 仓库名会成为你链接的一部分！
例如仓库名叫 claudecodeinstallguide，你的最终链接就是：
https://你的用户名.github.io/claudecodeinstallguide/
步骤 4：上传所有文件
  1. 创建仓库后，页面上会出现一行蓝色文字 "uploading an existing file（上传已有文件）"，点击它。
  2. 打开你电脑的 D:\claudeai 文件夹，全选以下文件：
    • index.html
    • claude_code_guide_v2.html
    • 全部 8 张图片（image_5b5c6f.png 等）
    • README.md（可选，显示在仓库首页的说明文字）
  3. 把选中的文件一起拖进浏览器窗口的虚线框内。
  4. 等待上传完成（文件较多可能需要 1-2 分钟）。
  5. 上传完成后，页面底部会出现一个绿色按钮，点击 Commit changes（保存提交）。

图9：上传成功后，仓库文件列表会显示你所有已上传的文件。确认文件齐全后，点击 Commit changes 保存即完成。

⚠️ 注意：如果你的仓库显示 Private（私有），朋友将无法访问链接。必须在创建时选择 Public（公开）。

步骤 5：开启网页服务并获取你的专属链接
  1. 回到仓库主页，点击顶部菜单栏的 Settings（设置） 齿轮图标。
  2. 在左侧菜单往下滑，找到并点击 Pages。
  3. 在 Build and deployment（构建与部署） 区域，找到 Branch（分支） 下拉菜单。
  4. 把下拉菜单从 "None" 改成 main，右边选 / (root)，然后点击 Save（保存）。
  5. 耐心等待约 1-2 分钟，然后刷新这个页面。
  6. 页面顶部会出现一个蓝色/绿色提示框，显示 "Your site is live at..."，后面跟着你的专属链接。

⚠️ 如果看到这个升级提示，说明你的仓库是私有的，GitHub Pages 无法启用。回到仓库设置把可见性改为 Public 即可解决。

图10：出现蓝色方框显示 "Your site is live at https://skyisthelimit111.github.io/claudecodeinstallguide/"，大功告成！复制这个链接发给朋友即可。

✅ 大功告成！把链接发给朋友的方式：
直接复制这个链接，通过微信、短信、邮件发给朋友。朋友无需安装任何软件，用手机或电脑浏览器打开就能查看完整教程！
